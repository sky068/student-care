# 学生托管系统需求设计

## 1. 背景与目标

本系统用于管理学生每日出勤和每日任务，核心场景包括：

- 家长维护学生每日任务。
- 家长维护学生每日上午、下午出勤情况。
- 教师查看学生每日任务，标记任务是否完成，并维护学生上午、下午出勤情况。
- 系统记录关键操作日志，方便追溯每日任务、出勤和完成状态的变更。

系统优先实现 Web 端，面向家长、教师和管理员三类用户。第一阶段以“按天管理”为核心，不做复杂排课、成绩统计和即时通讯。

## 2. 组织方式与注册流程

第一阶段采用“班级空间”作为核心组织方式。教师创建班级后，系统分别生成家长绑定码和教师邀请码；家长通过绑定码添加孩子，其他教师通过教师邀请码加入班级协同管理。两类邀请码隔离，避免身份越权。

### 2.1 注册身份

用户注册时必须选择初始身份：

- 教师
- 家长

同一普通账号可以同时开通教师、家长身份，并在两个工作台之间切换。身份切换只改变当前会话的权限上下文，不复制账号，也不合并教师班级关系和家长孩子关系。

- 教师身份只依据教师班级关系加载班级和学生。
- 家长身份只依据家长学生关系加载孩子。
- 班级、学生、孩子的最近选择按“用户 ID + 当前身份”分别保存。
- 管理员身份不与普通身份混用，也不能切换为教师或家长。
- 每次切换必须清空当前页面业务状态并按新身份重新请求数据，旧请求不得覆盖新工作台。
- 切换到尚无班级的教师身份或尚未绑定孩子的家长身份时，只展示对应空工作台，不自动跳转到创建、加入或绑定引导页；家长统一使用顶部已有的“绑定孩子”按钮，空工作台不重复显示入口。

第一阶段不开放普通用户自行注册管理员账号，管理员账号由系统初始化或后台创建。

注册主要字段：

- 手机号或登录账号
- 密码
- 姓名
- 身份：教师、家长

### 2.2 教师注册后流程

教师注册并登录成功后：

1. 如果教师还没有管理班级，系统引导创建班级或加入已有班级。
2. 创建班级时只需要填写班级名称，例如“一年级一班”。
3. 系统自动生成班级编号，例如 `A8K392`。
4. 创建班级的教师成为班级创建者 `owner`。
5. 其他教师可以输入独立的教师邀请码加入该班级，成为协同教师 `teacher`。
6. 教师进入班级后，可以查看和操作该班级下所有学生信息。

教师可管理多个班级，但第一阶段页面默认进入最近使用的班级，避免操作复杂。

### 2.3 多教师管理班级

一个班级可以由多个教师共同管理。

教师班级角色：

- `owner`：班级创建者。
- `teacher`：通过教师邀请码加入的协同教师。

第一阶段不做复杂审批。教师输入正确的教师邀请码后即可加入班级。所有教师都可以管理该班级学生、出勤和任务完成状态；家长绑定码不能用于教师加入。

### 2.4 家长注册后流程

家长注册并登录成功后：

1. 如果家长还没有绑定孩子，系统引导绑定孩子。
2. 家长输入教师提供的班级编号。
3. 家长填写孩子姓名。
4. 如果当前家长没有绑定过同班级、同姓名、同编号的孩子，系统创建一条新的学生记录并绑定当前家长。
5. 如果当前家长已经绑定过同班级、同姓名、同编号的孩子，系统不重复创建学生和绑定关系，并提示“该孩子已经绑定过”。
6. 如果其他家长绑定过同姓名、同编号的学生，系统不会自动复用该学生，避免其他家长获得该学生操作权限。

第一阶段不做复杂审批。家长输入正确班级编号后即可加入班级并绑定孩子。教师端提供移除学生、解绑家长和修改学生信息的能力，用于处理误绑定或重复绑定。

第一阶段不做自动合并。家长添加孩子时使用“家长用户 ID + 班级 ID + 学生姓名 + 备注”判断当前家长是否重复绑定；不同家长提交相同姓名和备注时，会生成独立学生记录。教师端负责识别和处理重复学生。

教师端学生列表需要对同班级内同姓名、同编号的学生自动增加展示后缀，便于核对：

- 第一条显示为 `张三 · 001`。
- 第二条显示为 `张三 · 001 (2)`。
- 第三条显示为 `张三 · 001 (3)`。

展示后缀只用于界面显示，不写入学生真实姓名字段。

### 2.5 班级邀请码规则

- 家长绑定码和教师邀请码均由系统生成，建议为 6 位大写字母和数字组合。
- 两类邀请码在系统内不可重复。
- 教师可以在班级设置页分别查看、复制两类邀请码。
- 班级创建者可以分别刷新或停用两类邀请码，且互不影响。
- 家长使用班级绑定码添加孩子，协同教师使用独立的教师邀请码加入，两个邀请码必须隔离，避免家长凭绑定码获得教师权限。

### 2.6 小程序迁移考虑

为了方便后续迁移到微信小程序，第一阶段接口和数据模型应避免依赖 Web 会话特性。

建议预留字段：

- 用户表预留 `wechat_openid`。
- 用户表预留 `wechat_unionid`。
- 登录态后续可从账号密码登录平滑切换到微信授权登录。

### 2.7 部署配置

第一阶段管理员账号不开放页面注册，部署时通过环境变量初始化。

建议环境变量：

- `PORT`：服务端口，默认 `3000`。
- `ADMIN_ACCOUNT`：管理员登录账号。
- `ADMIN_PASSWORD`：管理员登录密码。
- `ADMIN_NAME`：管理员显示名称，默认“系统管理员”。

服务启动时，如果配置了 `ADMIN_ACCOUNT` 和 `ADMIN_PASSWORD`：

- 当账号不存在时，系统自动创建管理员。
- 当账号已存在时，系统更新该账号为管理员角色，刷新密码，并启用账号。
- 如果只配置其中一个变量，系统跳过管理员初始化并输出警告。

## 3. 用户角色

### 3.1 家长

家长身份主要负责自己绑定学生的日常信息维护。

可执行操作：

- 查看已绑定学生列表。
- 查看学生某一天的上午、下午出勤状态。
- 设置或修改学生某一天上午、下午出勤状态。
- 新增学生某一天的任务。
- 修改自己创建且未被教师确认完成的任务。
- 删除自己创建且未被教师确认完成的任务。
- 查看任务完成状态。
- 查看和自己相关的操作记录。

限制：

- 只能操作自己绑定的学生。
- 不能标记任务完成。
- 不能修改教师已标记完成的任务完成状态。
- 不能查看其他家长绑定的学生。

### 3.2 教师

教师身份主要负责维护自己管理范围内学生的出勤情况，并确认任务完成情况。

可执行操作：

- 查看自己管理班级下的学生列表。
- 查看学生每日出勤状态。
- 修改学生每日的上午、下午出勤状态。
- 查看学生每日任务。
- 新增学生每日任务。
- 修改或删除自己创建且待完成的任务。
- 标记每条任务是否完成。
- 修改任务完成状态。
- 查看自己管理范围内学生的操作日志。
- 查看班级编号。
- 班级创建者可以刷新或停用班级编号。
- 移除班级内异常学生。
- 解绑错误的家长学生关系。

限制：

- 只能修改或删除自己创建且待完成的任务内容。
- 只能操作自己管理班级或学生范围内的数据。

### 3.3 管理员

管理员用于系统基础数据和异常处理。

可执行操作：

- 管理用户账号。
- 启用或停用用户账号。
- 查看全量班级、学生和家长绑定关系。
- 维护学生信息。
- 维护家长与学生绑定关系。
- 维护教师与班级、学生的管理关系。
- 查看全量操作日志。
- 必要时修正异常数据。

限制：

- 管理员操作必须记录日志。
- 管理员不参与日常任务完成标记，除非作为异常修正。

## 4. 核心业务对象

### 4.1 用户

用户是登录系统的主体。

主要字段：

- 用户 ID
- 登录账号
- 密码哈希
- 姓名
- 手机号
- 初始角色：家长、教师、管理员
- 已开通身份列表
- 最近使用身份
- 微信 OpenID
- 微信 UnionID
- 状态：启用、禁用
- 创建时间
- 更新时间

### 4.2 学生

学生是出勤和任务管理的对象。

主要字段：

- 学生 ID
- 姓名
- 性别
- 班级 ID
- 学号或内部编号
- 同班级同名学生组内固定托管编号
- 备注
- 状态：在读、停用
- 创建时间
- 更新时间

### 4.3 班级

班级用于组织学生和教师管理范围。

主要字段：

- 班级 ID
- 班级名称
- 班级编号
- 班级编号是否启用
- 年级
- 状态
- 创建教师 ID
- 创建时间
- 更新时间

### 4.4 家长学生关系

用于定义家长可以管理哪些学生。

主要字段：

- 关系 ID
- 家长用户 ID
- 学生 ID
- 关系类型：父亲、母亲、监护人、其他
- 是否主要联系人
- 绑定来源：班级编号加入、管理员绑定
- 创建时间

### 4.5 教师管理关系

用于定义教师可以查看和操作哪些学生或班级。

第一阶段建议采用教师和班级绑定。

主要字段：

- 关系 ID
- 教师用户 ID
- 班级 ID
- 角色：`owner`、`teacher`
- 创建时间

## 5. 出勤管理

### 5.1 出勤维度

每天分为两个出勤时段：

- 上午
- 下午

每个时段有一个出勤状态：

- 正常出勤
- 请假

### 5.2 出勤记录

每个学生每天应有一条出勤记录，记录上午和下午两个状态。

主要字段：

- 出勤记录 ID
- 学生 ID
- 日期
- 上午状态：正常出勤、请假
- 下午状态：正常出勤、请假
- 上午备注
- 下午备注
- 创建人 ID
- 创建时身份
- 最后修改人 ID
- 最后修改时身份
- 创建时间
- 更新时间

### 5.3 出勤规则

- 家长可以设置自己绑定学生的当天或指定日期出勤状态。
- 如果某天没有出勤记录，页面默认展示为空或默认正常出勤，具体实现建议默认展示为空，并在家长保存后生成记录。
- 家长修改出勤时，系统需要记录操作日志。
- 教师可以修改自己管理范围内学生的上午、下午出勤状态。
- 教师修改出勤时，系统需要记录操作日志。
- 同一学生同一日期只能有一条出勤记录。

### 5.4 出勤操作场景

家长设置出勤：

1. 家长登录系统。
2. 选择学生。
3. 选择日期。
4. 设置上午状态和下午状态。
5. 可填写备注。
6. 保存。
7. 系统写入或更新出勤记录。
8. 系统记录操作日志。

教师设置出勤：

1. 教师登录系统。
2. 选择班级和日期。
3. 查看学生列表及上午、下午出勤状态。
4. 修改指定学生的上午状态和下午状态。
5. 可填写备注。
6. 保存。
7. 系统写入或更新出勤记录。
8. 系统记录操作日志。

## 6. 每日任务管理

### 6.1 任务来源

任务可由家长或教师创建，归属于某个学生和某一天。任务需要明确保留创建人和创建身份。

### 6.2 任务状态

任务完成状态包括：

- 待完成
- 已完成

默认状态为待完成。

### 6.3 任务记录

主要字段：

- 任务 ID
- 学生 ID
- 日期
- 任务标题
- 任务内容
- 完成状态：待完成、已完成
- 创建人 ID
- 创建时身份
- 最后修改人 ID
- 最后修改时身份
- 完成标记人 ID
- 完成标记时身份
- 完成标记时间
- 教师批注人 ID、批注时身份和批注时间
- 创建时间
- 更新时间
- 删除标记

### 6.4 任务规则

- 家长只能给自己绑定的学生添加每日任务，教师只能给自己管理范围内的学生添加每日任务。
- 家长和教师只能修改或删除同一账号以当前身份创建且状态为待完成的任务；切换身份后不能管理另一身份创建的任务。
- 已完成任务不能直接修改或删除，需要教师先将状态改回待完成。
- 教师可以标记自己管理范围内学生的任务完成状态。
- 教师可以通过下拉框在待完成和已完成之间切换，系统需要记录日志。
- 删除任务建议采用软删除，保留历史记录和日志追溯能力。
- 同一天同一学生可以有多条任务。

### 6.5 任务操作场景

家长或教师新增任务：

1. 用户登录系统。
2. 选择学生。
3. 选择日期。
4. 输入任务标题和任务内容。
5. 保存。
6. 系统创建任务，默认待完成。
7. 系统记录操作日志。

教师标记任务完成：

1. 教师登录系统。
2. 选择班级和日期。
3. 查看学生任务列表。
4. 对单条任务通过下拉框选择待完成或已完成。
5. 系统更新任务完成状态。
6. 系统记录操作日志。

## 7. 操作日志

### 7.1 日志目标

操作日志用于追踪关键业务数据变化，包括：

- 谁操作了数据。
- 操作了哪个学生。
- 操作了哪一天的数据。
- 操作类型是什么。
- 操作前后内容是什么。
- 操作发生时间。

### 7.2 日志范围

第一阶段需要记录以下操作：

- 家长新增任务。
- 家长修改任务。
- 家长删除任务。
- 教师新增任务。
- 教师修改任务。
- 教师删除任务。
- 家长新增出勤记录。
- 家长修改出勤记录。
- 教师新增出勤记录。
- 教师修改出勤记录。
- 教师标记任务已完成。
- 教师标记任务待完成。
- 教师创建班级。
- 教师通过教师邀请码加入班级。
- 教师分别刷新或停用家长绑定码、教师邀请码。
- 家长通过班级编号绑定孩子。
- 教师移除学生或解绑家长。
- 管理员维护用户、学生、绑定关系。
- 普通账号开通第二身份或切换当前身份。

### 7.3 日志字段

主要字段：

- 日志 ID
- 操作用户 ID
- 操作用户角色
- 操作类型
- 业务对象类型：出勤、任务、用户、学生、班级、绑定关系
- 业务对象 ID
- 学生 ID
- 日期
- 操作前数据
- 操作后数据
- 客户端 IP
- User-Agent
- 操作时间

### 7.4 日志展示

家长可查看：

- 自己操作产生的日志。
- 自己绑定学生相关的任务和出勤日志。

教师可查看：

- 自己管理学生范围内的任务和出勤日志。

管理员可查看：

- 全部日志。
- 按日期查询 15 至 180 天内的归档日志。
- 日志列表直接展示任务标题、学生和业务日期，业务对象 ID 仅作为辅助信息。
- 点击日志可查看操作前后的数据快照；任务删除或改名后仍以日志快照保持可读。

### 7.5 日志保留与归档

- 最近 14 天日志保留在日常数据中，供工作台快速查询。
- 15 至 180 天日志迁移至独立归档表，不参与日常数据加载。
- 超过 180 天的日志自动删除。
- 系统启动时检查归档状态，运行期间每天最多执行一次归档维护。

## 8. 页面规划

### 8.1 登录与注册页

功能：

- 注册时选择教师或家长身份。
- 用户登录。
- 根据角色进入不同首页。

### 8.2 教师班级引导页

功能：

- 教师首次登录时创建班级。
- 教师首次登录时通过教师邀请码加入已有班级。
- 展示家长绑定码和教师邀请码。
- 提供复制两类邀请码的入口。
- 支持分别刷新或停用两类邀请码。

### 8.3 家长绑定孩子页

功能：

- 家长首次登录时输入班级编号。
- 填写孩子姓名。
- 创建当前家长自己的学生记录，或识别当前家长已绑定的同名同编号学生。
- 家长工作台的孩子信息展示班级和备注；学生编号保留在教师、管理员管理场景中使用。
- 家长可在孩子信息中修改备注；后端必须校验家长绑定关系并记录修改前后内容和操作人。具有该学生访问权限的教师、管理员也可通过接口维护备注。
- 绑定完成后进入家长端首页。

### 8.4 家长端首页

功能：

- 展示绑定学生。
- 快速选择日期。
- 查看所选学生当天上午、下午出勤状态。
- 查看所选学生当天任务列表。
- 通过“新建任务”按钮打开弹窗新增任务。
- 修改出勤。

### 8.5 家长任务管理页

功能：

- 筛选条件在手机端采用单列布局，查询、刷新等主要操作使用完整宽度或明确的两列按钮布局。
- 每条记录的主要身份信息、状态和高频操作应在首屏可见；技术 ID 作为次要信息展示并允许换行。
- 启用、停用、移除、解绑等危险操作必须与普通操作拉开距离，并在执行前二次确认。
- 日志详情、密码修改、绑定关系等弹窗在手机端使用底部面板或全宽弹层，内容区域可独立滚动且关闭入口始终可见。

## 10. 数据约束

- 用户账号唯一。
- 同一普通账号最多拥有教师、家长两个身份；管理员身份必须独立。
- 家长绑定码和教师邀请码在系统内不可重复。
- 任一邀请码停用后只影响对应加入流程，不得影响另一类邀请码。
- 同一教师不能重复加入同一个班级。
- 学生姓名允许重复；优先通过学生编号区分，没有学生编号时使用班级内固定托管编号。
- 托管编号在同班级同名学生组内从 01 开始分配，创建后保持不变，学生移除后不回收、不重新排序。
- 不同家长提交相同班级、姓名、备注时，系统不自动复用已有学生。
- 同一家长不能重复绑定同班级、同姓名、同备注的孩子，重复操作时提示已绑定。
- 同班级重名学生全部显示区分标识；学生编号重复或缺失时追加固定托管编号。
- 同一学生同一日期只能有一条出勤记录。
- 任务必须绑定学生和日期。
- 任务、出勤和操作日志必须记录操作时身份；权限不能只比较用户 ID。
- 升级时为旧账号补齐单元素身份列表，并按原始账号角色回填历史任务、出勤的操作身份字段；迁移必须可重复执行。
- 家长操作学生数据前必须校验绑定关系。
- 教师操作任务完成状态前必须校验教师管理范围。
- 教师操作学生出勤前必须校验教师管理范围。
- 学生被移除后，家长和教师均不得通过旧 ID 继续访问其出勤、任务或日志。
- 删除任务不能物理删除，必须保留软删除记录。
- 所有关键写操作必须写操作日志。
- 基于全量快照持久化的写操作必须串行执行，避免并发请求相互覆盖。

## 11. 非功能需求

### 11.1 安全

- 密码必须使用不可逆安全哈希存储，不能明文保存。
- 本地开发命令可监听局域网以支持手机真机调试；生产启动仍应默认仅监听本机，并通过受控的部署配置显式开放公网访问。
- 正式环境密码必须通过 HTTPS 传输；无域名的公网 IP 明文访问仅作为显式开启的临时部署模式，并应通过安全组限制来源。密码存储使用带随机盐、可版本升级的慢哈希，旧参数在成功登录后渐进升级。
- 后端接口必须校验角色和数据范围，不能只依赖前端隐藏按钮。
- 身份切换接口必须校验账号已开通目标身份；所有后续接口以服务端会话中的当前身份鉴权。
- 日志中的敏感字段需要脱敏或避免记录。
- 非管理员日志响应不得返回 IP、User-Agent 或操作前后完整快照，也不得返回无关班级或用户的日志。
- 禁用用户不能登录。
- 登录会话有效期为 30 天；用户被停用或密码重置后，现有会话立即失效。
- 登录和注册接口必须按来源地址限流；单个 JSON 请求体不得超过 64KB，所有枚举、日期和文本长度由后端校验。
- 登录后的写操作按账号与来源地址限流；教师最多管理 50 个班级，家长最多绑定 50 个有效孩子，每名学生每天最多保留 100 条未删除任务，防止公开注册账号无限消耗服务器资源。
- API 响应必须设置 `Cache-Control: no-store`；服务端内部异常只向客户端返回统一错误文案，详细堆栈仅写入服务端日志。
- 邀请码不能作为唯一权限依据，家长绑定学生后仍需通过家长学生关系校验数据权限。

### 11.2 可用性

- 日期选择需要支持快速切换今天、前一天、后一天。
- 教师首次登录后应直接引导创建班级。
- 教师首次登录后也可以直接输入教师邀请码加入已有班级。
- 家长首次登录后应直接引导绑定孩子。
- 家长在绑定孩子页面可以返回工作台或暂不添加，不应被迫只能完成绑定或退出登录。
- 系统应按登录用户记住教师上次选择的班级和学生，以及家长上次选择的孩子；选择失效时自动回退到当前可用项。
- 普通账号应在工作台和首次引导页提供身份切换入口；未开通第二身份时可明确确认后开通。
- 教师任务列表需要支持按完成状态筛选。
- 出勤状态需要清晰区分上午和下午。
- 出勤表单按上午、下午分为两行；每行将出勤状态与对应备注并排展示，移动端不得产生横向滚动。
- 任务完成操作应支持快速点击切换。

#### 11.2.1 移动端优先适配规则

- 所有页面均先以 320～430 像素宽度设计和验收，再扩展到平板与桌面端；管理员页面不得例外。
- 移动端必须保留完整业务能力，不得以隐藏功能代替适配。
- 页面结构在窄屏下采用单列信息流，固定侧栏应转换为顶部导航、抽屉或可折叠区域。
- 系统品牌标题字号不得小于当前工作台标题，品牌层级应在手机端保持清晰。
- 工具栏、筛选器和表单在手机端应自动换行或切换为单列，不能依赖横向挤压维持桌面布局。
- 工作台日期、刷新和退出操作在 320 像素以上视口优先保持同一行，同时不得缩小 44 像素的最小触控高度。
- 日期输入框与日历按钮必须作为共享边框的组合控件展示，日历图标固定在日期输入框右侧，不得因工具栏压缩而偏向相邻操作按钮。
- 桌面端日期组合控件应预留完整日期文本宽度，自定义日历按钮启用时隐藏原生图标并释放其占位；日期不得被截断。
- 可点击区域最小尺寸为 44×44 像素，相邻操作之间应保留至少 8 像素间距。
- 输入框字号不得小于 16 像素，避免 iOS Safari 聚焦时自动缩放；日期、搜索和账号字段应使用正确的输入类型。
- 不得依赖鼠标悬停完成关键操作，所有功能必须支持触控和键盘操作，并提供清晰的焦点状态。
- 弹窗最大高度应基于 `100svh`，内容超出时内部滚动，并兼容顶部、底部安全区域。
- 新建任务在手机端使用底部面板、桌面端使用居中弹窗；任务列表页面不长期占用空间展示创建表单。
- 新建或编辑任务时，触屏设备和窄屏端不得自动聚焦输入框或弹出软键盘，焦点应落在关闭按钮以确保用户先看到完整任务面板；桌面端默认聚焦任务标题，编辑时选中原标题。任务弹窗应支持 Escape 关闭、焦点约束及关闭后恢复触发点；打开期间锁定底层页面及其滚动位置，只允许弹窗内容独立纵向滚动。
- 长姓名、账号、任务标题和技术 ID 必须支持合理换行或省略，不得撑破页面；省略的信息应能在详情中完整查看。
- 状态不能仅依赖颜色区分，应同时提供文字；文本与背景需要保持足够对比度。
- 移动端页面主体不得出现非预期横向滚动；确需横向滚动的表格、导航应限制在自身容器内。
- 学生或孩子选择器的搜索框和选项列表必须锁定在容器宽度内，只允许纵向滚动，不得响应横向拖动。
- 家长端已有孩子时，孩子选择器与“绑定孩子”按钮在手机端保持同一行；下拉菜单按整行宽度展开。暂无孩子时绑定按钮使用完整宽度。
- 至少验证 320、375、390、430、768 像素视口，并覆盖 iOS Safari 与 Android Chrome 的点击、输入、日期选择、弹窗和滚动行为。

### 11.3 可追溯

- 出勤、任务和权限维护操作必须记录日志。
- 日志保留 180 天，其中最近 14 天为热数据，其余为归档数据。

### 11.4 性能

- 第一阶段使用 SQLite 单文件数据库，默认路径为 `data/stumng.sqlite`。
- SQLite 便于部署和后续迁移到其他关系型数据库。
- SQLite 文件不是加密存储，部署时需要限制服务器文件权限并做好备份。
- 教师端按班级和日期查询时，应避免一次加载过长时间范围的数据。
- 归档日志按操作时间建立索引，超过 180 天后自动清理，避免数据库无限膨胀。

### 11.5 质量保障

- 业务日期统一按 `Asia/Shanghai` 计算，并严格验证 `YYYY-MM-DD` 对应真实日历日期。
- 工作台异步加载需要防止旧响应覆盖当前班级、学生或日期的数据。
- 教师批注自动保存必须串行，后输入内容不能被较早请求覆盖。
- 自动化集成测试至少覆盖并发写入、学生移除后的权限、日志隔离、邀请码隔离、非法日期和超大请求。

## 12. 建议接口草案

接口路径仅作为后续实现参考。

### 12.1 认证

- `POST /api/auth/register`：注册，选择教师或家长身份。
- `POST /api/auth/login`：登录。
- `POST /api/auth/logout`：退出登录。
- `GET /api/auth/me`：获取当前用户信息。
- `POST /api/auth/roles`：为当前普通账号开通教师或家长身份。
- `POST /api/auth/switch-role`：切换当前会话身份，并记住最近使用身份。

登录后的前端请求携带 `X-Active-Role` 当前身份标识；该标识必须与服务端会话身份一致，不一致时返回 `409`，防止切换前发出的迟到请求以新身份执行。

### 12.2 用户

- `GET /api/users`：管理员查看用户列表。
- `PATCH /api/users/{userId}/status`：管理员启用或停用用户。
- `PATCH /api/users/{userId}/password`：管理员修改用户密码；成功后使该用户的现有登录会话失效。

限制：

- 普通注册接口不能创建管理员。
- 管理员账号通过部署环境变量初始化。
- 管理员不能停用当前登录的管理员账号。
- 新注册、管理员重置和部署管理员密码统一要求至少 8 位，且必须包含大写字母、小写字母和特殊符号。提示需明确指出缺少的条件。已有旧密码继续允许登录，成功登录后仅升级哈希参数，不强制中断用户。操作日志不得记录密码或密码哈希。

### 12.3 班级

- `POST /api/classes`：教师创建班级。
- `POST /api/classes/join-by-code`：教师通过教师邀请码加入已有班级。
- `GET /api/classes`：获取当前教师管理的班级列表。
- `GET /api/classes/{classId}`：获取班级详情。
- `PATCH /api/classes/{classId}`：修改班级名称。
- `POST /api/classes/{classId}/code/refresh`：刷新班级编号。
- `POST /api/classes/{classId}/code/disable`：停用班级编号。
- `POST /api/classes/{classId}/teacher-code/refresh`：刷新教师邀请码。
- `POST /api/classes/{classId}/teacher-code/disable`：停用教师邀请码。

### 12.4 学生

- `GET /api/students`：获取当前用户可访问的学生列表。
- `POST /api/students/bind-by-class-code`：家长通过班级编号绑定孩子。
- `PATCH /api/students/{studentId}/status`：管理员移除或恢复学生。
- `GET /api/classes/{classId}/students`：教师或管理员查看班级学生。
- `DELETE /api/classes/{classId}/students/{studentId}`：教师移除异常学生。
- `POST /api/parent-student-relations`：管理员通过家长账号和学生 ID 新增绑定关系。
- `GET /api/parent-student-relations`：管理员查看家长学生绑定关系。
- `DELETE /api/parent-student-relations/{relationId}`：教师或管理员解除错误的家长学生绑定。

### 12.5 出勤

- `GET /api/attendance?studentId=&date=`：查询某学生某日出勤。
- `PUT /api/attendance`：新增或更新某学生某日出勤。

### 12.6 任务

- `GET /api/tasks?studentId=&date=`：查询某学生某日任务。
- `POST /api/tasks`：家长或教师新增任务。
- `PUT /api/tasks/{taskId}`：家长或教师修改自己创建且待完成的任务。
- `DELETE /api/tasks/{taskId}`：家长或教师删除自己创建且待完成的任务。
- `PATCH /api/tasks/{taskId}/completion`：教师切换任务状态，支持 `pending`、`completed`。
- `PATCH /api/tasks/{taskId}/teacher-remark`：教师维护家长可见的任务批注。

### 12.7 日志

- `GET /api/operation-logs?limit=&offset=`：分页查询最近 14 天操作日志。
- `GET /api/operation-logs/archive?startDate=&endDate=&limit=&offset=`：管理员按日期分页查询归档日志，单次最多返回 100 条。

说明：

- 家长只能查看自己或绑定学生相关日志。
- 教师只能查看管理范围内学生相关日志。
- 管理员可以查看全量日志。

## 14. 第一阶段交付范围

建议第一阶段实现：

- 注册、登录和角色识别。
- 教师创建班级并生成班级编号。
- 教师通过教师邀请码加入已有班级。
- 教师查看家长绑定码和教师邀请码。
- 班级创建者分别刷新或停用两类邀请码。
- 家长通过班级编号绑定孩子。
- 家长查看绑定学生。
- 家长维护每日出勤。
- 教师维护管理范围内学生的每日出勤。
- 家长新增、修改、删除每日任务。
- 教师查看班级学生每日任务，并新增、修改或删除自己创建且待完成的任务。
- 教师标记任务完成状态。
- 教师移除异常学生或解绑错误绑定。
- 管理员通过环境变量初始化账号。
- 管理员查看用户列表。
- 管理员启用或停用用户账号。
- 管理员查看班级列表、班级编号、加入状态和教师列表。
- 管理员查看学生列表、所属班级、绑定家长和重复学生标记。
- 管理员移除或恢复学生。
- 管理员查看、新增和解除家长学生绑定关系。
- 管理员查看全量操作日志。
- 操作日志记录和基础查询。

暂不实现：

- 微信或短信通知。
- 复杂请假审批流程。
- 成绩管理。
- 课程表管理。
- 多校区组织架构。
- 文件上传。
- 移动端 App。
- 复杂加入班级审批。
- 复杂组织架构和多级权限。
