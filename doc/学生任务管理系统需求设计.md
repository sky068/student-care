# 学生任务管理系统需求设计

## 1. 背景与目标

本系统用于管理学生每日出勤和每日任务，核心场景包括：

- 家长维护学生每日任务。
- 家长维护学生每日上午、下午出勤情况。
- 教师查看学生每日任务，标记任务是否完成，并维护学生上午、下午出勤情况。
- 系统记录关键操作日志，方便追溯每日任务、出勤和完成状态的变更。

系统优先实现 Web 端，面向家长、教师和管理员三类用户。第一阶段以“按天管理”为核心，不做复杂排课、成绩统计和即时通讯。

## 2. 组织方式与注册流程

第一阶段采用“班级空间”作为核心组织方式。教师创建班级，系统生成班级编号；家长通过班级编号绑定孩子并加入班级；其他教师也可以通过同一个班级编号加入班级协同管理。这样流程简单，也便于后续迁移到微信小程序。

### 2.1 注册身份

用户注册时必须选择身份：

- 教师
- 家长

第一阶段不开放普通用户自行注册管理员账号，管理员账号由系统初始化或后台创建。

注册主要字段：

- 手机号或登录账号
- 密码
- 姓名
- 身份：教师、家长

### 2.2 教师注册后流程

教师注册并登录成功后：

1. 如果教师还没有管理班级，系统引导创建班级或加入已有班级。
2. 创建班级时只需要填写班级名称，例如“一年级一班”。
3. 系统自动生成班级编号，例如 `A8K392`。
4. 创建班级的教师成为班级创建者 `owner`。
5. 其他教师可以输入班级编号加入该班级，成为协同教师 `teacher`。
6. 教师进入班级后，可以查看和操作该班级下所有学生信息。

教师可管理多个班级，但第一阶段页面默认进入最近使用的班级，避免操作复杂。

### 2.3 多教师管理班级

一个班级可以由多个教师共同管理。

教师班级角色：

- `owner`：班级创建者。
- `teacher`：通过班级编号加入的协同教师。

第一阶段不做复杂审批。教师输入正确班级编号后即可加入班级。为了保持权限简单，所有教师都可以管理该班级学生、出勤和任务完成状态，但只有班级创建者可以刷新或停用班级编号。

### 2.4 家长注册后流程

家长注册并登录成功后：

1. 如果家长还没有绑定孩子，系统引导绑定孩子。
2. 家长输入教师提供的班级编号。
3. 家长填写孩子姓名。
4. 如果当前家长没有绑定过同班级、同姓名、同编号的孩子，系统创建一条新的学生记录并绑定当前家长。
5. 如果当前家长已经绑定过同班级、同姓名、同编号的孩子，系统不重复创建学生和绑定关系，并提示“该孩子已经绑定过”。
6. 如果其他家长绑定过同姓名、同编号的学生，系统不会自动复用该学生，避免其他家长获得该学生操作权限。

第一阶段不做复杂审批。家长输入正确班级编号后即可加入班级并绑定孩子。教师端提供移除学生、解绑家长和修改学生信息的能力，用于处理误绑定或重复绑定。

第一阶段不做自动合并。学生绑定使用“家长用户 ID + 班级 ID + 学生姓名 + 学生编号”判断当前家长是否重复绑定；不同家长提交相同姓名和编号时，会生成独立学生记录。教师端负责识别和处理重复学生。

教师端学生列表需要对同班级内同姓名、同编号的学生自动增加展示后缀，便于核对：

- 第一条显示为 `张三 · 001`。
- 第二条显示为 `张三 · 001 (2)`。
- 第三条显示为 `张三 · 001 (3)`。

展示后缀只用于界面显示，不写入学生真实姓名字段。

### 2.5 班级编号规则

- 班级编号由系统生成，建议为 6 位大写字母和数字组合。
- 班级编号在系统内唯一。
- 教师可以在班级设置页查看班级编号。
- 班级创建者可以刷新班级编号，刷新后旧编号失效。
- 班级创建者可以停用班级编号，停用后家长和协同教师不能通过该编号加入。
- 家长和教师使用同一个班级编号加入，系统根据当前登录身份决定加入方式。

### 2.6 小程序迁移考虑

为了方便后续迁移到微信小程序，第一阶段接口和数据模型应避免依赖 Web 会话特性。

建议预留字段：

- 用户表预留 `wechat_openid`。
- 用户表预留 `wechat_unionid`。
- 登录态后续可从账号密码登录平滑切换到微信授权登录。

### 2.7 部署配置

第一阶段管理员账号不开放页面注册，部署时通过环境变量初始化。

建议环境变量：

- `PORT`：服务端口，默认 `3000`。
- `ADMIN_ACCOUNT`：管理员登录账号。
- `ADMIN_PASSWORD`：管理员登录密码。
- `ADMIN_NAME`：管理员显示名称，默认“系统管理员”。

服务启动时，如果配置了 `ADMIN_ACCOUNT` 和 `ADMIN_PASSWORD`：

- 当账号不存在时，系统自动创建管理员。
- 当账号已存在时，系统更新该账号为管理员角色，刷新密码，并启用账号。
- 如果只配置其中一个变量，系统跳过管理员初始化并输出警告。

## 3. 用户角色

### 3.1 家长

家长身份主要负责自己绑定学生的日常信息维护。

可执行操作：

- 查看已绑定学生列表。
- 查看学生某一天的上午、下午出勤状态。
- 设置或修改学生某一天上午、下午出勤状态。
- 新增学生某一天的任务。
- 修改自己创建且未被教师确认完成的任务。
- 删除自己创建且未被教师确认完成的任务。
- 查看任务完成状态。
- 查看和自己相关的操作记录。

限制：

- 只能操作自己绑定的学生。
- 不能标记任务完成。
- 不能修改教师已标记完成的任务完成状态。
- 不能查看其他家长绑定的学生。

### 3.2 教师

教师身份主要负责维护自己管理范围内学生的出勤情况，并确认任务完成情况。

可执行操作：

- 查看自己管理班级下的学生列表。
- 查看学生每日出勤状态。
- 修改学生每日的上午、下午出勤状态。
- 查看学生每日任务。
- 新增学生每日任务。
- 修改或删除自己创建且待完成的任务。
- 标记每条任务是否完成。
- 修改任务完成状态。
- 查看自己管理范围内学生的操作日志。
- 查看班级编号。
- 班级创建者可以刷新或停用班级编号。
- 移除班级内异常学生。
- 解绑错误的家长学生关系。

限制：

- 只能修改或删除自己创建且待完成的任务内容。
- 只能操作自己管理班级或学生范围内的数据。

### 3.3 管理员

管理员用于系统基础数据和异常处理。

可执行操作：

- 管理用户账号。
- 启用或停用用户账号。
- 查看全量班级、学生和家长绑定关系。
- 维护学生信息。
- 维护家长与学生绑定关系。
- 维护教师与班级、学生的管理关系。
- 查看全量操作日志。
- 必要时修正异常数据。

限制：

- 管理员操作必须记录日志。
- 管理员不参与日常任务完成标记，除非作为异常修正。

## 4. 核心业务对象

### 4.1 用户

用户是登录系统的主体。

主要字段：

- 用户 ID
- 登录账号
- 密码哈希
- 姓名
- 手机号
- 角色：家长、教师、管理员
- 微信 OpenID
- 微信 UnionID
- 状态：启用、禁用
- 创建时间
- 更新时间

### 4.2 学生

学生是出勤和任务管理的对象。

主要字段：

- 学生 ID
- 姓名
- 性别
- 班级 ID
- 学号或内部编号
- 备注
- 状态：在读、停用
- 创建时间
- 更新时间

### 4.3 班级

班级用于组织学生和教师管理范围。

主要字段：

- 班级 ID
- 班级名称
- 班级编号
- 班级编号是否启用
- 年级
- 状态
- 创建教师 ID
- 创建时间
- 更新时间

### 4.4 家长学生关系

用于定义家长可以管理哪些学生。

主要字段：

- 关系 ID
- 家长用户 ID
- 学生 ID
- 关系类型：父亲、母亲、监护人、其他
- 是否主要联系人
- 绑定来源：班级编号加入、管理员绑定
- 创建时间

### 4.5 教师管理关系

用于定义教师可以查看和操作哪些学生或班级。

第一阶段建议采用教师和班级绑定。

主要字段：

- 关系 ID
- 教师用户 ID
- 班级 ID
- 角色：`owner`、`teacher`
- 创建时间

## 5. 出勤管理

### 5.1 出勤维度

每天分为两个出勤时段：

- 上午
- 下午

每个时段有一个出勤状态：

- 正常出勤
- 请假

### 5.2 出勤记录

每个学生每天应有一条出勤记录，记录上午和下午两个状态。

主要字段：

- 出勤记录 ID
- 学生 ID
- 日期
- 上午状态：正常出勤、请假
- 下午状态：正常出勤、请假
- 上午备注
- 下午备注
- 创建人 ID
- 最后修改人 ID
- 创建时间
- 更新时间

### 5.3 出勤规则

- 家长可以设置自己绑定学生的当天或指定日期出勤状态。
- 如果某天没有出勤记录，页面默认展示为空或默认正常出勤，具体实现建议默认展示为空，并在家长保存后生成记录。
- 家长修改出勤时，系统需要记录操作日志。
- 教师可以修改自己管理范围内学生的上午、下午出勤状态。
- 教师修改出勤时，系统需要记录操作日志。
- 同一学生同一日期只能有一条出勤记录。

### 5.4 出勤操作场景

家长设置出勤：

1. 家长登录系统。
2. 选择学生。
3. 选择日期。
4. 设置上午状态和下午状态。
5. 可填写备注。
6. 保存。
7. 系统写入或更新出勤记录。
8. 系统记录操作日志。

教师设置出勤：

1. 教师登录系统。
2. 选择班级和日期。
3. 查看学生列表及上午、下午出勤状态。
4. 修改指定学生的上午状态和下午状态。
5. 可填写备注。
6. 保存。
7. 系统写入或更新出勤记录。
8. 系统记录操作日志。

## 6. 每日任务管理

### 6.1 任务来源

任务可由家长或教师创建，归属于某个学生和某一天。任务需要明确保留创建人和创建身份。

### 6.2 任务状态

任务完成状态包括：

- 待完成
- 已完成

默认状态为待完成。

### 6.3 任务记录

主要字段：

- 任务 ID
- 学生 ID
- 日期
- 任务标题
- 任务内容
- 完成状态：待完成、已完成
- 创建人 ID
- 最后修改人 ID
- 完成标记人 ID
- 完成标记时间
- 创建时间
- 更新时间
- 删除标记

### 6.4 任务规则

- 家长只能给自己绑定的学生添加每日任务，教师只能给自己管理范围内的学生添加每日任务。
- 家长和教师只能修改或删除自己创建且状态为待完成的任务。
- 已完成任务不能直接修改或删除，需要教师先将状态改回待完成。
- 教师可以标记自己管理范围内学生的任务完成状态。
- 教师可以通过下拉框在待完成和已完成之间切换，系统需要记录日志。
- 删除任务建议采用软删除，保留历史记录和日志追溯能力。
- 同一天同一学生可以有多条任务。

### 6.5 任务操作场景

家长或教师新增任务：

1. 用户登录系统。
2. 选择学生。
3. 选择日期。
4. 输入任务标题和任务内容。
5. 保存。
6. 系统创建任务，默认待完成。
7. 系统记录操作日志。

教师标记任务完成：

1. 教师登录系统。
2. 选择班级和日期。
3. 查看学生任务列表。
4. 对单条任务通过下拉框选择待完成或已完成。
5. 系统更新任务完成状态。
6. 系统记录操作日志。

## 7. 操作日志

### 7.1 日志目标

操作日志用于追踪关键业务数据变化，包括：

- 谁操作了数据。
- 操作了哪个学生。
- 操作了哪一天的数据。
- 操作类型是什么。
- 操作前后内容是什么。
- 操作发生时间。

### 7.2 日志范围

第一阶段需要记录以下操作：

- 家长新增任务。
- 家长修改任务。
- 家长删除任务。
- 教师新增任务。
- 教师修改任务。
- 教师删除任务。
- 家长新增出勤记录。
- 家长修改出勤记录。
- 教师新增出勤记录。
- 教师修改出勤记录。
- 教师标记任务已完成。
- 教师标记任务待完成。
- 教师创建班级。
- 教师通过班级编号加入班级。
- 教师刷新或停用班级编号。
- 家长通过班级编号绑定孩子。
- 教师移除学生或解绑家长。
- 管理员维护用户、学生、绑定关系。

### 7.3 日志字段

主要字段：

- 日志 ID
- 操作用户 ID
- 操作用户角色
- 操作类型
- 业务对象类型：出勤、任务、用户、学生、班级、绑定关系
- 业务对象 ID
- 学生 ID
- 日期
- 操作前数据
- 操作后数据
- 客户端 IP
- User-Agent
- 操作时间

### 7.4 日志展示

家长可查看：

- 自己操作产生的日志。
- 自己绑定学生相关的任务和出勤日志。

教师可查看：

- 自己管理学生范围内的任务和出勤日志。

管理员可查看：

- 全部日志。

## 8. 页面规划

### 8.1 登录与注册页

功能：

- 注册时选择教师或家长身份。
- 用户登录。
- 根据角色进入不同首页。

### 8.2 教师班级引导页

功能：

- 教师首次登录时创建班级。
- 教师首次登录时通过班级编号加入已有班级。
- 展示班级编号。
- 提供复制班级编号入口。
- 支持刷新或停用班级编号。

### 8.3 家长绑定孩子页

功能：

- 家长首次登录时输入班级编号。
- 填写孩子姓名。
- 创建当前家长自己的学生记录，或识别当前家长已绑定的同名同编号学生。
- 绑定完成后进入家长端首页。

### 8.4 家长端首页

功能：

- 展示绑定学生。
- 快速选择日期。
- 查看所选学生当天上午、下午出勤状态。
- 查看所选学生当天任务列表。
- 新增任务。
- 修改出勤。

### 8.5 家长任务管理页

功能：

- 按学生和日期查看任务。
- 新增任务。
- 修改待完成任务。
- 删除待完成任务。
- 查看任务完成状态。

### 8.6 家长出勤管理页

功能：

- 按学生和日期维护上午、下午出勤。
- 填写请假备注。
- 查看最近出勤记录。

### 8.7 教师端首页

功能：

- 展示管理班级。
- 支持切换班级。
- 展示当前班级编号。
- 按班级和日期查看学生列表。
- 查看每个学生出勤情况。
- 修改每个学生上午、下午出勤情况。
- 查看任务完成概览。
- 新增学生任务。

### 8.8 教师出勤管理页

功能：

- 按班级、学生、日期筛选出勤记录。
- 维护学生上午、下午出勤状态。
- 填写或修改出勤备注。
- 查看最近出勤记录。

### 8.9 教师任务确认页

功能：

- 按班级、学生、日期筛选任务。
- 新增任务。
- 修改或删除自己创建且待完成的任务。
- 通过下拉框切换单条任务状态：待完成、已完成。
- 查看任务创建人和创建时间。

### 8.10 班级设置页

功能：

- 修改班级名称。
- 查看、刷新或停用班级编号。
- 查看协同教师列表。
- 查看学生列表。
- 移除异常学生。
- 解绑错误的家长学生关系。

### 8.11 日志页

功能：

- 按学生、日期、操作人、操作类型筛选。
- 展示操作摘要。
- 查看操作前后数据。

### 8.12 管理后台

功能：

- 用户管理：查看用户列表，启用或停用账号。
- 班级管理：查看班级编号、加入状态和教师列表。
- 学生管理：查看学生、班级、家长和重复学生标记，支持移除和恢复学生。
- 家长学生绑定：查看绑定关系，解除错误绑定，支持管理员手动新增绑定关系。
- 全量日志查询：查看系统关键操作日志。

第一阶段管理后台定位为基础运维和异常处理，不提供复杂批量导入、批量编辑、学生合并和完整用户资料编辑。

## 10. 数据约束

- 用户账号唯一。
- 班级编号在系统内唯一。
- 班级编号停用后不能用于家长绑定孩子，也不能用于协同教师加入班级。
- 同一教师不能重复加入同一个班级。
- 学生姓名允许重复，必要时通过学生编号或备注区分。
- 不同家长提交相同班级、姓名、学生编号时，系统不自动复用已有学生。
- 同一家长不能重复绑定同班级、同姓名、同学生编号的孩子，重复操作时提示已绑定。
- 教师端对同班级内同姓名、同学生编号的学生自动生成展示后缀和“需核对”标记。
- 同一学生同一日期只能有一条出勤记录。
- 任务必须绑定学生和日期。
- 家长操作学生数据前必须校验绑定关系。
- 教师操作任务完成状态前必须校验教师管理范围。
- 教师操作学生出勤前必须校验教师管理范围。
- 删除任务不能物理删除，必须保留软删除记录。
- 所有关键写操作必须写操作日志。

## 11. 非功能需求

### 11.1 安全

- 密码必须加密存储，不能明文保存。
- 后端接口必须校验角色和数据范围，不能只依赖前端隐藏按钮。
- 日志中的敏感字段需要脱敏或避免记录。
- 禁用用户不能登录。
- 班级编号不能作为唯一权限依据，家长绑定学生后仍需通过家长学生关系校验数据权限。

### 11.2 可用性

- 日期选择需要支持快速切换今天、前一天、后一天。
- 教师首次登录后应直接引导创建班级。
- 教师首次登录后也可以直接输入班级编号加入已有班级。
- 家长首次登录后应直接引导绑定孩子。
- 系统应按登录用户记住教师上次选择的班级和学生，以及家长上次选择的孩子；选择失效时自动回退到当前可用项。
- 教师任务列表需要支持按完成状态筛选。
- 出勤状态需要清晰区分上午和下午。
- 任务完成操作应支持快速点击切换。

### 11.3 可追溯

- 出勤、任务和权限维护操作必须记录日志。
- 日志至少保留 180 天，具体保留周期后续可配置。

### 11.4 性能

- 第一阶段使用 SQLite 单文件数据库，默认路径为 `data/stumng.sqlite`。
- SQLite 便于部署和后续迁移到其他关系型数据库。
- SQLite 文件不是加密存储，部署时需要限制服务器文件权限并做好备份。
- 教师端按班级和日期查询时，应避免一次加载过长时间范围的数据。
- 操作日志表后续可按时间建立索引或归档。

## 12. 建议接口草案

接口路径仅作为后续实现参考。

### 12.1 认证

- `POST /api/auth/register`：注册，选择教师或家长身份。
- `POST /api/auth/login`：登录。
- `POST /api/auth/logout`：退出登录。
- `GET /api/auth/me`：获取当前用户信息。

### 12.2 用户

- `GET /api/users`：管理员查看用户列表。
- `PATCH /api/users/{userId}/status`：管理员启用或停用用户。
- `PATCH /api/users/{userId}/password`：管理员修改用户密码；成功后使该用户的现有登录会话失效。

限制：

- 普通注册接口不能创建管理员。
- 管理员账号通过部署环境变量初始化。
- 管理员不能停用当前登录的管理员账号。
- 新密码长度为 6 至 128 位，操作日志不得记录密码或密码哈希。

### 12.3 班级

- `POST /api/classes`：教师创建班级。
- `POST /api/classes/join-by-code`：教师通过班级编号加入已有班级。
- `GET /api/classes`：获取当前教师管理的班级列表。
- `GET /api/classes/{classId}`：获取班级详情。
- `PATCH /api/classes/{classId}`：修改班级名称。
- `POST /api/classes/{classId}/code/refresh`：刷新班级编号。
- `POST /api/classes/{classId}/code/disable`：停用班级编号。

### 12.4 学生

- `GET /api/students`：获取当前用户可访问的学生列表。
- `POST /api/students/bind-by-class-code`：家长通过班级编号绑定孩子。
- `PATCH /api/students/{studentId}/status`：管理员移除或恢复学生。
- `GET /api/classes/{classId}/students`：教师或管理员查看班级学生。
- `DELETE /api/classes/{classId}/students/{studentId}`：教师移除异常学生。
- `POST /api/parent-student-relations`：管理员通过家长账号和学生 ID 新增绑定关系。
- `GET /api/parent-student-relations`：管理员查看家长学生绑定关系。
- `DELETE /api/parent-student-relations/{relationId}`：教师或管理员解除错误的家长学生绑定。

### 12.5 出勤

- `GET /api/attendance?studentId=&date=`：查询某学生某日出勤。
- `PUT /api/attendance`：新增或更新某学生某日出勤。

### 12.6 任务

- `GET /api/tasks?studentId=&date=`：查询某学生某日任务。
- `POST /api/tasks`：家长或教师新增任务。
- `PUT /api/tasks/{taskId}`：家长或教师修改自己创建且待完成的任务。
- `DELETE /api/tasks/{taskId}`：家长或教师删除自己创建且待完成的任务。
- `PATCH /api/tasks/{taskId}/completion`：教师切换任务状态，支持 `pending`、`completed`。

### 12.7 日志

- `GET /api/operation-logs`：查询操作日志。

说明：

- 家长只能查看自己或绑定学生相关日志。
- 教师只能查看管理范围内学生相关日志。
- 管理员可以查看全量日志。

## 13. 建议数据库表

第一阶段使用 SQLite。当前实现为了保持代码简单，使用一张通用记录表保存各业务集合；后续如果数据量增长，可以平滑拆分成标准关系型表。

建议后续拆分表：

- `users`
- `students`
- `classes`
- `parent_student_relations`
- `teacher_class_relations`
- `attendance_records`
- `daily_tasks`
- `operation_logs`

关键索引建议：

- `users.account`
- `users.wechat_openid`
- `classes.class_code`
- `students.class_id`
- `parent_student_relations.parent_user_id`
- `parent_student_relations.student_id`
- `teacher_class_relations.teacher_user_id`
- `attendance_records.student_id + attendance_date`
- `daily_tasks.student_id + task_date`
- `operation_logs.student_id + operation_time`
- `operation_logs.operator_user_id + operation_time`

## 14. 第一阶段交付范围

建议第一阶段实现：

- 注册、登录和角色识别。
- 教师创建班级并生成班级编号。
- 教师通过班级编号加入已有班级。
- 教师查看班级编号。
- 班级创建者刷新或停用班级编号。
- 家长通过班级编号绑定孩子。
- 家长查看绑定学生。
- 家长维护每日出勤。
- 教师维护管理范围内学生的每日出勤。
- 家长新增、修改、删除每日任务。
- 教师查看班级学生每日任务，并新增、修改或删除自己创建且待完成的任务。
- 教师标记任务完成状态。
- 教师移除异常学生或解绑错误绑定。
- 管理员通过环境变量初始化账号。
- 管理员查看用户列表。
- 管理员启用或停用用户账号。
- 管理员查看班级列表、班级编号、加入状态和教师列表。
- 管理员查看学生列表、所属班级、绑定家长和重复学生标记。
- 管理员移除或恢复学生。
- 管理员查看、新增和解除家长学生绑定关系。
- 管理员查看全量操作日志。
- 操作日志记录和基础查询。

暂不实现：

- 微信或短信通知。
- 复杂请假审批流程。
- 成绩管理。
- 课程表管理。
- 多校区组织架构。
- 文件上传。
- 移动端 App。
- 复杂加入班级审批。
- 复杂组织架构和多级权限。
